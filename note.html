Transaction Detection

ONNX-based photo scanning to auto-detect transaction images from today's gallery photos.

13components shipped
5steps in user flow
3items on backlog

✓ What was done

  • Dependencies added: onnxruntime_v2, image, photo_manager, workmanager
  • Model asset: assets/models/teacher.onnx registered in pubspec.yaml
  • TransactionClassifier (lib/core/services/transaction_classifier.dart): ONNX inference wrapper using EfficientNet-B2, threshold 0.5 (high-recall mode for suggestions)
  • PhotoScannerService (lib/core/services/photo_scanner_service.dart): Fetches photos for a date from device gallery via photo_manager, runs each through classifier
  • DetectionResult model (lib/core/models/detection_result.dart): Holds AssetEntity + confidence score
  • DetectionProvider (lib/providers/detection.dart): Resumable state management — supports pause/resume, streams results as they're found, caches processed photo IDs so scanning continues from where it left off
  • TodayDetectionScreen (lib/screens/today/today_detection_screen.dart): Replaced the old Transactions tab. Compact approval card layout — 72px thumbnail, time, confidence badge, approve/dismiss actions. Swipe-to-dismiss. Scanning auto-starts when tab is active, pauses when leaving
  • DetectionResultsScreen (lib/screens/detection/detection_results_screen.dart): Full-screen view for the settings test button (custom date scanning)
  • Settings screen: Date picker + scan button for testing, daily schedule toggle + time picker
  • Tab lifecycle: MainTabsScreen pauses detection on tab-away, resumes on tab-return. IndexedStack keeps widget alive

→ Today Tab Flow

  1. User taps the second tab (sparkle icon)
  2. Detection starts automatically for today's photos
  3. Results stream in as compact tiles while scanning continues
  4. User swipes left to dismiss, taps check to approve a detection
  5. Navigating away pauses scanning — returning resumes from cache

⏱ Daily Background Scan Job

  • workmanager package added for cross-platform background task scheduling
  • ScanScheduleService (lib/core/services/scan_schedule_service.dart): Hive-backed settings for enabled/disabled state and scheduled time (default 10:00 PM). Registers/cancels WorkManager periodic task with correct initial delay calculation
  • Background callback (lib/core/services/background_scan.dart): Top-level entry point that loads the ONNX model, scans today's photos, and stores detected results in a scan_results Hive box
  • Settings UI: Toggle to enable/disable daily scan + time picker to adjust schedule
  • Platform setup: iOS BGTaskSchedulerPermittedIdentifiers in Info.plist, Android READ_MEDIA_IMAGES permission in AndroidManifest

✓ Preprocessing — Letterbox

  • Pipeline: scale longest side to 320 (aspect ratio preserved) → pad to 320×320 with ImageNet mean RGB (124, 116, 104) → normalize → CHW tensor
  • No crop: full screenshot content retained; matches training get_eval_transforms letterbox
